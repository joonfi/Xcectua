# Governance Diff Classification

## Purpose
Classify governance‑relevant changes to determine
required evidence and scrutiny level.

Classification does **not** affect enforcement.

---

## Types

### TYPE‑0 — No Governance Impact
- Non‑governance files
- Pure documentation
- Formatting only

### TYPE‑1 — Editorial Governance Changes
- Comments
- Clarifications
- No logic impact

### TYPE‑2 — Logical Governance Changes
- Process changes
- Evidence rules
- Governance tooling behavior

### TYPE‑3 — High‑Risk Governance Changes
- Enforcement‑adjacent logic
- Validator‑related changes
- Requires heightened scrutiny (DISF)

---

## Rules

- Classification is descriptive, not permissive.
- Higher types require more evidence.
- Classification never bypasses validation.

---

## Authority

Classification informs **process**.
Validation remains exclusive to `rexce`.
